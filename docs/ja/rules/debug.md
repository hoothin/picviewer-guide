# デバッグ

ルールデバッグの目標は、「ルールがヒットしたか」と「返されたURLが正しいか」を確認することです。

## デバッグの有効化

設定で `debug` を有効にすると、ブラウザのコンソールにより多くのマッチング情報が出力されます。

## トラブルシューティング手順

1. `url` がターゲットサイトと一致するか確認します。
2. `src` / `lazyAttr` がヒットするか確認します。
3. `r/s` の順序と正規表現を確認します。
4. `xhr` を使用する場合は、セレクターとリンクを確認します。
5. 複雑なロジックの場合は、`getImage` / `getExtSrc` 内にログを追加します。

## 一般的な問題

- 置換後もサムネイルのまま：`r/s` のマッチングが不十分か、順序が間違っています。
- ルールが有効にならない：`url` が広すぎるか狭すぎます。
- 内部ページ取得の失敗：セレクターが古いか、クロスオリジン制限があります。
